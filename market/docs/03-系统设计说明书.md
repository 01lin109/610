# 商城后台管理系统 - 系统设计说明书

## 1. 系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────┐
│              Nginx (反向代理)                 │
├─────────────────────────────────────────────┤
│   market_frontend (Vue3 + Vite)             │
│   静态资源服务 / SPA 路由                     │
├─────────────────────────────────────────────┤
│   market_backend (Spring Boot)              │
│   RESTful API 服务 :8080                    │
│   ├── Controller 层 (REST接口)              │
│   ├── Service 层 (业务逻辑)                  │
│   ├── Mapper 层 (数据访问)                   │
│   └── Security (JWT认证+权限控制)            │
├─────────────────────────────────────────────┤
│   MySQL :3306 (数据持久化)                   │
│   Redis :6379 (缓存/Token)                  │
└─────────────────────────────────────────────┘
```

### 1.2 分层架构

| 层级 | 职责 | 技术 |
|------|------|------|
| 表现层 | RESTful API接口，参数校验 | Spring MVC Controller |
| 业务层 | 业务逻辑处理，事务管理 | Spring Service |
| 持久层 | 数据库 CRUD 操作 | MyBatis Mapper |
| 基础设施层 | 安全认证、配置、工具类 | Spring Security / JWT |

## 2. 核心模块设计

### 2.1 安全认证模块

- **认证流程**：登录 → 验证密码 → 生成JWT → 返回Token
- **鉴权流程**：请求携带Token → JwtAuthenticationFilter解析 → 加载用户权限 → SecurityConfig校验
- **密码加密**：BCryptPasswordEncoder
- **Token有效期**：24小时

### 2.2 统一返回结果

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

- 成功：code=200
- 业务异常：code=400
- 未授权：code=401
- 系统异常：code=500

### 2.3 分页查询设计

请求参数：
```json
{
  "page": 1,
  "pageSize": 10,
  "keyword": "搜索关键词"
}
```

响应格式：
```json
{
  "code": 200,
  "data": {
    "records": [...],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 2.4 全局异常处理

| 异常类型 | HTTP状态码 | 触发场景 |
|---------|-----------|---------|
| BusinessException | 400 | 业务规则校验失败 |
| MethodArgumentNotValidException | 400 | 参数校验失败 |
| AccessDeniedException | 403 | 权限不足 |
| AuthenticationException | 401 | 未登录或Token过期 |
| Exception | 500 | 未知系统异常 |

### 4.2 状态管理（Pinia）

- userStore：存储用户信息和Token，提供 login / logout / getUserInfo action
- 登录成功后Token存储至localStorage
- 路由守卫检测Token，未登录跳转至/login

### 4.3 Axios封装

- baseURL: `/api`
- 请求拦截器：自动注入Token
- 响应拦截器：统一处理401跳转登录页
- 超时时间：15秒
